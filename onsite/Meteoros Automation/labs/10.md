# Biggest Problem Faced

Project creation kept failing again and again.

Error looked like this:

```txt id="q1tx3d"
A project can't be created because of a conflicting policy in your organization's subscription.
```

Sometimes also got:

```txt id="j14hfx"
Copying... Please try again
```

At one point I honestly thought:

* resource creation failed
* Azure AI Foundry broke
* region was wrong
* hub creation failed

---

# Things I Tried

| Attempt                | Result |
| ---------------------- | ------ |
| Changing project names | Failed |
| Changing hub names     | Failed |
| Using random names     | Failed |
| Changing regions       | Failed |
| Retrying creation      | Failed |

---

> [!WARNING]
> Azure sandbox environments can behave weirdly.
> Sometimes backend resources get created even when frontend UI shows failure.

---

# Actual Root Cause

Later opened Azure Portal manually and checked Resource Groups.

Found out these resources were ALREADY created:

| Resource          | Status          |
| ----------------- | --------------- |
| `project61997769` | Already Created |
| `hub61997769`     | Already Created |
| Foundry Resource  | Already Created |

So the UI was misleading the whole time 😭

The backend resources existed successfully but Foundry frontend kept asking:

* “Create Project”

This was the major debugging point in the lab.

---

# Important Learning

Instead of trusting UI blindly:

✅ Check:

* Azure Portal
* Resource Groups
* Existing AI resources
* Backend-created services

This actually saved the lab.
