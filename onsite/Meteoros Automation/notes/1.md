# Why Passwords Are Hidden in CLI / Terminals

## Question

Why doesn’t CLI/terminal show passwords while typing in tools like:

* WSL
* Azure CLI
* Linux terminal
* SSH
* Git
* Docker

---

## Short Answer

Because terminals temporarily disable:

```text
Terminal Echo
```

which means:

```text
Keyboard Input --> Application
```

works,

but:

```text
Application --> Screen
```

is blocked.

So the password is still being typed, just not displayed.

---

> [!NOTE]
> This is standard Unix/Linux terminal behavior.

---

# What is Terminal Echo?

Normally:

```text
Keyboard --> Terminal --> Screen
```

Example:

You type:

```text
hello
```

Terminal shows:

```text
hello
```

This display behavior is called:

```text
Terminal Echo
```

---

# During Password Input

Applications do:

```c
disable_terminal_echo();
read_password();
enable_terminal_echo();
```

So:

| Action                     | Happens? |
| -------------------------- | -------- |
| Keyboard input received    | ✅ Yes    |
| Characters shown on screen | ❌ No     |

---

# Why Hide Passwords?

If passwords were visible:

```text
mypassword123
```

people nearby could steal them through:

* shoulder surfing
* screen sharing
* recordings
* screenshots

---

> [!CAUTION]
> Visible passwords = direct security risk.

---

# Why Not Show `*****` Instead?

They could, but that also leaks information.

Example:

```text
************
```

reveals:

```text
Password Length = 12
```

It may also expose:

* typing speed
* pauses
* typing patterns

This is called:

```text
Keystroke Dynamics
```

---

# Why No Eye 👁 Icon Like Websites?

CLI terminals are:

* text-based
* keyboard-driven
* minimal UI systems

Eye icons need:

* graphics
* mouse interaction
* richer UI rendering

Traditional terminals were never designed for that.

Also CLI tools must work everywhere:

* SSH
* remote servers
* containers
* recovery terminals

---

# Is It Bad UX?

For beginners:

* yes, sometimes confusing

For security systems:

* intentional tradeoff

---

> [!IMPORTANT]
> CLI tools prioritize security + minimal exposure over beginner-friendly UI.

---

# Quick Summary

| Thing                         | Reason                        |
| ----------------------------- | ----------------------------- |
| Password invisible            | Terminal echo disabled        |
| No `*****`                    | Avoid password length leakage |
| No 👁 icon                    | CLI is minimal text UI        |
| Typing still works            | Input is captured internally  |
| Common in Linux/WSL/Azure CLI | Standard Unix behavior        |
